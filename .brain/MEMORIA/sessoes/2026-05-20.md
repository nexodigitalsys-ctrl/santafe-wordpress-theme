# Sessão 2026-05-20 — Revisão Mobile Completa

## Problemas Encontrados

### 1. 🔴 CRÍTICO — Conteúdo invisível em mobile
**Descrição:** Todo o conteúdo abaixo do hero ficava completamente escuro/invisível em viewports mobile.

**Causa:** `IntersectionObserver` no `premium-interactions.js` usava `threshold: 0.15` com `rootMargin: '0px 0px -50px 0px'`, que não disparava para seções grandes em telas pequenas. O fallback de 3s funcionava, mas era péssima UX.

**Solução:**
- `threshold: 0.05` (mais sensível)
- `rootMargin: '50px 0px 50px 0px'` (mais margem)
- Fallback reduzido de `3000ms` para `1500ms`
- Adicionado `will-change: opacity, transform` para performance
- CSS fallback no `header.php`: `[data-reveal]` sempre visível por padrão, `.js` classe controla animação
- `<html class="no-js">` → script inline remove e adiciona `js`

### 2. 🟡 MÉDIO — Sticky CTA cobrindo formulário de contato
**Descrição:** O sticky CTA mobile (WhatsApp + Llamar) ficava sobre o formulário de contato, dificultando o preenchimento.

**Solução:** Adicionado `pb-20 lg:pb-0` no `<body>` para garantir padding inferior em mobile, compensando a altura do sticky CTA.
